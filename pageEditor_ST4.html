2. Technical Description of Implemented Pages

User Profile Page (profile.html)

HTML Structure

The page utilizes semantic HTML5 tags to ensure a logical structure. A <header> contains the logo and main <nav>. The primary content is wrapped in a <main> tag, split into functional containers using <div> elements (e.g., .profile-container, .column). The progressive output relies on hidden <div> elements containing unordered lists (<ul>) and <span> tags serving as injection points for DOM manipulation.

CSS Techniques

I utilized CSS Flexbox for the main layout to create a responsive, two-column structure (flex-wrap: wrap). To align with the "Life on Land" theme, I implemented a "glassmorphism" aesthetic using backdrop-filter: blur(20px) and linear gradients to create frosted, translucent profile cards. Interactive elements feature smooth CSS transitions (transform: scale(1.02) on hover) to improve user feedback.

Design and Implementation Rationale

The layout is designed to separate the user's avatar/basic info from the interactive "Profile Builder" to maintain a clean interface. I opted to use JavaScript prompt() sequences rather than traditional HTML forms to create a guided, conversational experience that reduces cognitive load. JavaScript handles state tracking via a profileData object and dynamically updates the DOM and progress bar to provide immediate visual feedback.

4. Challenges and Lessons Learned

Challenge: Managing the state of the progressive profile builder and updating the DOM accurately without refreshing the page.
Resolution: I structured the user data into a centralized JavaScript object (profileData) and wrote a dedicated updateDOM() function. By tracking the exact number of filled prompts out of the total, I could accurately calculate the progress bar percentage.
Lesson Learned: I learned how to effectively use the FileReader API to process local image uploads and display them directly in the browser using base64 data URLs without needing a backend server.

5. Accessibility Considerations

Accessibility was prioritized throughout the development process. Specific techniques include:

  • Semantic Structure: Using <header>, <main>, and logical heading hierarchies (h1 to h3) to allow screen readers to parse the document efficiently.
  • Alternative Text: All images, including the dynamic avatar and website logo, have descriptive alt attributes.
  • ARIA Labels: Added aria-label="Main Navigation" to the <nav> to clarify its purpose for assistive technologies.
  • Contrast & Readability: The glassmorphism backgrounds were carefully tuned with adequate opacity to ensure the dark text remains highly readable against the background gradient.
