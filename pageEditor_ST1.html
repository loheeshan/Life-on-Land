Home page

Technical Description

The Home page was built using semantic HTML elements such as header, nav, main, section, article, and footer to create a clear and meaningful page structure. The hero section was used to introduce the website topic with a heading, short description, buttons, and an image panel. A card-based layout was used to present key SDG 15 topics such as land restoration, biodiversity, and local action.

For styling, I used separate CSS files for global styles, navigation, footer, and page-specific design. Flexbox was mainly used to align the hero content and card layout. Key CSS properties include display: flex, gap, padding, margin, border-radius, and object-fit. These were used to create a responsive and visually clean design.

Accessibility

Accessibility was addressed by including a skip link to allow keyboard users to jump directly to the main content. All images include descriptive alt text, and the navigation uses a clear structure with meaningful link names. The page also uses responsive design so the layout works well on different screen sizes.

Gallery Page

Technical Description

The Gallery page was created to display SDG-related images in a structured and interactive way. The main image area uses a grid layout, while each image includes a data-description attribute to store text that is displayed dynamically in the popup. JavaScript was used to create the lightbox effect, allowing users to open, close, and move between images.

CSS Grid was used for the gallery image layout because it makes it easy to organise images evenly. The popup overlay was styled using properties such as position: fixed, display, background-color, border-radius, and transition. Additional controls were added so users can change the popup background colour and description font style.

Accessibility

All gallery images include descriptive alt text. The popup buttons and controls are clearly labelled and easy to identify. The page uses a simple layout and readable structure, which helps users interact with the gallery more easily across different devices.

Content Page

Technical Description

The Content page was designed using a two-column structure with a sidebar and a main content area. The sidebar uses internal links so users can jump directly to specific sections of the page. The main content is divided into separate article blocks, each focusing on one topic such as restoration, student action, community impact, and long-term thinking.

Flexbox was used to create the page layout, while card-style content sections were styled using padding, margin-bottom, border, and background-color. The image rows were also styled using Flexbox. A highlight box and a quote box were added to make important information stand out without making the layout too complex.

Accessibility

The page includes a skip link and a back-to-top button to improve navigation. Images include descriptive alt text, and the heading structure was adjusted to improve accessibility and validation. Internal navigation links also make the page easier to use, especially for longer content.

Challenges and Lessons Learned

One challenge was separating CSS into different files while still keeping the design consistent across all pages. I solved this by creating separate files for global styles, navigation, footer, and page-specific layouts. Another challenge was balancing good design with simplicity, especially because the project also needs to be explained in the viva.

I also faced validation and accessibility issues such as heading warnings and missing structure. These were resolved by improving heading hierarchy, adding hidden headings where needed, and checking the pages again after corrections. This helped me understand the importance of semantic structure and accessibility.

Compliance

The pages I created follow good web development practice by using semantic HTML, clear navigation, descriptive links, and accessible content. Images include alt text, the pages are responsive, and the layout avoids misleading or harmful content. I also tried to keep the site readable and usable across different devices.

In relation to JANET-style guidance discussed in lectures, the work is educational, relevant to the coursework, and avoids inappropriate or non-academic material. While there is still room for small improvements, the pages generally follow good standards for structure, usability, and responsible publishing.
